# Chrome ウェブストア申請フロー・審査対策ガイド

## 申請フロー（実際に外部公開する際の手順）

1. **Chrome Web Store Developer Dashboard に登録**（未登録の場合）
   https://chrome.google.com/webstore/devconsole/ — 初回登録時に一度だけ $5 の登録料が必要です
   （Googleアカウントでの支払いが必要なため、これは開発者ご自身で行ってください）。
2. 「新しいアイテムを追加」→ `release/clipkeep-for-notion-1.0.0.zip` をアップロード
3. 「ストアの掲載情報」タブに [`LISTING.md`](LISTING.md) の内容を転記
   - 短い説明・詳細な説明・カテゴリ（生産性）・言語
   - スクリーンショット4枚（`store/screenshots/`）をアップロード
   - ストアアイコン（128×128、`public/icons/icon128.png`。[[公開前チェックリスト]]の
     アイコン品質の項目を参照）
4. 「プライバシーに関する取り組み」タブ
   - プライバシーポリシーURL（[`PRIVACY_POLICY.md`](PRIVACY_POLICY.md)を外部公開したURL）を入力
   - データ利用状況の開示チェックボックス（後述「データ利用開示の注意点」を参照）
   - 権限ごとの正当性説明（[`LISTING.md`](LISTING.md)の表の内容を転記）
5. 「対象範囲」で公開範囲（全世界 / 特定の国のみ）や公開タイプ（公開 / 限定公開 / 非公開）を選択
6. 審査に提出。審査期間は通常数時間〜数日（権限の種類や初回審査かどうかで変動）

## データ利用開示の注意点（つまずきやすいポイント）

「プライバシーに関する取り組み」タブの「このアイテムが処理するユーザーデータ」チェックボックスは、
**実際の挙動と正確に一致させる必要があります**。ClipKeepの場合:

- ✅ **Website content（ウェブサイトのコンテンツ）**: チェックが必要
  （クリップ対象ページの本文を読み取り、Notionへ送信するため）
- ✅ **Authentication information（認証情報）**: チェックが必要
  （Notionインテグレーショントークンを保存・送信するため）
- ❌ Personally identifiable information、位置情報、健康情報、金融情報、個人間のやり取り等: 該当なし
  （現状のMVPでは収集していません）
- 用途は **「App functionality（アプリの機能提供）」** のみを選択し、「広告」「分析」等は選択しないこと
- 「このデータを第三者に販売しない」「本来の目的以外に利用しない」という趣旨の証明チェックボックスに
  正直にチェックできる状態を保つこと（現状のMVPはこれを満たしています）

この開示内容と実際の権限・通信先が食い違っていると、審査で **最も指摘されやすいポイント** になります。

## 審査で指摘されやすいその他のポイント

| 項目 | 内容 | ClipKeepの現状 |
|---|---|---|
| 権限の過剰要求 | 使っていない権限や`<all_urls>`のような広範な権限を要求していないか | ✅ `activeTab`方式で最小限。問題なし |
| 単一目的の逸脱 | 説明文と実際の機能が単一目的に収まっているか | ✅ 「Notionへ保存する」で一貫 |
| プライバシーポリシーの不備 | URLが無効/内容が実際の挙動と矛盾 | ✅ 公開済み: https://nobu1120.github.io/clipkeeper/privacy-policy.html （[[公開前チェックリスト]]） |
| 商標の扱い | 拡張機能名やアイコンで他社ブランドを無断使用していないか | ⚠️ 「Notion」を名称に含む（[[公開前チェックリスト]]で判断が必要） |
| 動作しない機能の表示 | ボタンや機能があるのに実際には機能しない（deceptive functionality） | ✅ 対応済み — Pro/ライセンスキーUIを削除し、無料機能のみで完結する構成に変更（[[公開前チェックリスト]]） |
| リモートコード実行 | `eval`や外部から読み込むスクリプトがないか | ✅ CSPで`script-src 'self'`のみ許可、該当なし |
| Manifest V3準拠 | MV2の記法が残っていないか | ✅ MV3で実装済み |

## 審査提出後

- 差し戻された場合は、Dashboardに具体的な違反ポリシー名と該当箇所が通知されるので、該当ドキュメント
  （本ガイド・[`LISTING.md`](LISTING.md)・[`PRIVACY_POLICY.md`](PRIVACY_POLICY.md)）を修正し再提出する
- 公開後にpermissionsやhost_permissionsを追加する変更は、再度同様の審査対象になる
