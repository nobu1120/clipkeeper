## 3. 残タスク・要判断事項（公開前に決めるべきこと）

### 3-1. 決済連携・OAuthの要否 → **v1では不要と判断（推奨）**

- 現状Proプラン/ライセンスキーは**ローカル検証のみのスタブ**で、実際の決済導線（Stripe等）は未実装
- 決済・サーバーサイドのライセンス検証・Notion Public Integration（OAuth）への移行は、
  「初回リリースをまず無料版として出し、需要を見てから有料化する」方針であれば**v1では不要**
- Notion認証も現状の「Internal Integration Token手動貼り付け」方式のままで初回公開は可能
  （ユーザーがNotion側でトークンを発行する一手間はあるが、機能上は問題なく動作する）
- 決済を先に実装したい場合は、[`SPEC.md`](SPEC.md)の非スコープ項目を参照して別途スコープを組む

### 3-2. Pro/ライセンスキーUIの扱い（⚠️ 判断が必要・下記「承認事項」参照）

現在オプション画面に「ライセンスキーを適用」ボタンと入力欄が表示されますが、これは
**実際には販売されていない**キーを検証するローカルスタブです。「開発中」という文言は
入れていますが、実際に押せるボタン・入力欄が公開版に残ったままだと:

- ユーザーが「本当に買えるのか」と誤解する可能性がある
- Chrome Web Storeの審査で「動作しない機能（deceptive functionality）」として指摘される可能性がある
- 日本国内向けであれば、実体のない有料プランを匂わせる表示は景品表示法上の考え方とも関わりうる
  （実際に対価を取っているわけではないため直ちに違反とは言えませんが、誤認を招く表示は避けるべき）

**推奨**: 公開版では一時的に「Proプランは近日公開予定です」という非活性表示に変え、
実際に押せる入力欄・ボタンは決済導線ができるまで隠す。

→ この変更を実施するかどうかは[[承認事項]]の一つとしています（実装自体は数分で可能です）。

### 3-3. 拡張機能名の商標リスク（⚠️ 判断が必要・下記「承認事項」参照）

拡張機能名に「Notion」を含む「ClipKeep **for Notion**」を使用しています。

- 同種の非公式クリッパー（"Save to Notion"等）が実在し、Chrome Web Storeで公開・審査通過している
  前例はあります（一般的に「◯◯ for X」という互換性を表す名称は許容されることが多い）
- ただし最終的な可否はGoogleの審査およびNotion社の商標方針次第であり、100%安全とは言い切れません
- リスクを避けたい場合は、名称から「Notion」を外し（例:「ClipKeep — Reliable Web Clipper」）、
  説明文中で「Notion対応」と述べる形に変更することも可能です

→ このまま申請するか、名称を変更するかは[[承認事項]]の一つとしています。

### 3-4. ストアアイコンの品質（法的・審査上の必須事項ではないが推奨）

現在のアイコンはプレースホルダー（単色の丸、`scripts/make-icons.mjs`で機械生成）です。
審査は通る可能性が高いですが、ストアでの見栄え・信頼感の観点からは公開前にきちんとした
ロゴへ差し替えることを推奨します。

### 3-5. プライバシーポリシーのホスティング（外部公開が必要・要承認）

[`store/PRIVACY_POLICY.md`](store/PRIVACY_POLICY.md)はMarkdown草案です。Chrome Web Storeには
**Web上でアクセス可能なURL**として登録する必要があるため、どこかに公開する必要があります
（GitHub Pages、note、個人サイトなど）。これは外部公開行為に当たるため、[[承認事項]]としています。

### 3-6. 連絡先メールアドレスの確定（個人情報の公開判断・要承認）

プライバシーポリシー内の連絡先は現在プレースホルダーです。公開する場合、その情報が
世界中に公開されることになるため、どのメールアドレスを載せるかご確認ください。
